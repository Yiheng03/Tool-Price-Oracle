# 价格驱动因素矩阵

用于系统检查金属 D+1 波动判断前容易遗漏的驱动因素。

## 七类驱动

| 维度 | 检查内容 |
|------|----------|
| 供应端 | 矿山、冶炼、港口、物流、出口配额 |
| 国内政策 | 国储、环保限产、战略矿产、出口管制 |
| 国际资金 | LME 仓单、CFTC、SHFE 持仓、升贴水 |
| 季节性 | 雨季、汛期、春节、消费旺季 |
| 替代效应 | 材料替代、技术路线变化、下游替换 |
| 内外价差 | SHFE-LME、进口盈亏、区域升贴水 |
| 需求端 | PMI、机床、汽车、基建、电子、能源 |

## 使用流程

1. 输入金属代码和日期。
2. 对七类驱动逐项检查。
3. 标注每一项的方向、强度、时效和证据。
4. 输出给金属行情分析师作为 D+1 判断补充。

## 输出格式

```markdown
## 价格驱动因素检查 [{metal}] [{date}]

| 维度 | 当前状态 | 方向 | 强度 | 证据 |
|------|----------|------|------|------|
| 供应端 | {状态} | ↑/↓/→ | 高/中/低 | {source} |

### 综合判断

- D+1 方向倾向：看涨 / 看跌 / 震荡
- 幅度建议：{range}
- 置信度：高 / 中 / 低
- 关键不确定性：{uncertainty}
```

## 检查原则

- 只使用能追溯来源的现货、新闻、库存、政策和需求信息。
- 多个驱动相互抵消时，输出震荡并说明原因。
- 重大政策或突发供应事件优先级高于常规季节性。
